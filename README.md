# unraid-nix

A native, 100% Rust-backed plugin to run the Nix package manager and manage Nix Flakes natively on Unraid without Docker container or VM overhead.

---

## 🚀 Quick Installation

To install this plugin on your Unraid server:

1. Log in to your Unraid WebGUI.
2. Navigate to the **Plugins** tab.
3. Click on **Install Plugin**.
4. Paste the following URL into the input field:
   ```text
   https://raw.githubusercontent.com/UberMetroid/unraid-nix/main/nix.plg
   ```
5. Click **Install** and wait for the installer to finish.

## 1. How It Works (Unraid RAM/Persistence Split)

Unraid runs its root operating system entirely in RAM, rebuilding it on every boot from your flash drive. To work within this architecture safely:

1. **Persistent Files (Flash Drive):**
   * Path: `/boot/config/plugins/nix/`
   * Stores settings (`nix.cfg`), the service orchestration file (`process-compose.yml`), and the plugin package manager (`nix.plg`).
2. **RAM Runtime Files (RAMFS):**
   * Path: `/usr/local/emhttp/plugins/nix/`
   * Wiped on reboot. The `nix.plg` installer extracts the WebGUI templates and the compiled Rust helper (`nix-helper`) back here on boot.
3. **The Nix Store (Storage Array/Cache):**
   * Path: `/nix` (bind-mounted to your persistent fast pool, e.g., `/mnt/cache/system/nix`).
   * Contains your installed packages, channels, and operational files. Nothing is lost on reboot.

## 2. Logs & Debugging (Where to look)

To make troubleshooting easy, logs are divided into distinct files at every layer of the execution stack:

| Log Target | File Location | Purpose |
| :--- | :--- | :--- |
| **Unraid System Logs** | `syslog` (View via top-right GUI button) | Boot installation, mounting operations, and init scripts (`logger -t nix-plugin`). |
| **Rust Helper Logs** | `/var/log/nix-helper.log` | Errors or outputs generated by the Rust backend CLI (`nix-helper`). |
| **Service Supervisor** | `/var/log/nix-process-compose.log` | Status changes and API events for the `process-compose` supervisor. |
| **Individual Flakes** | `/var/log/nix-services/<service-name>.log` | Live console output for each running service (e.g. Radarr, Jellyfin). |

### Real-Time Log Monitoring:
To watch your Nix services boot up in real time, open the Unraid terminal and run:
```bash
tail -f /var/log/nix-process-compose.log /var/log/nix-services/*.log
```

## 3. Sandboxing & Share Security

Nix services run in a lightweight **`bubblewrap`** sandbox. This isolates the process filesystem, ensuring it cannot access your disks or flash drive unless you explicitly map them under **Path Mappings** in the Settings:

```bash
# Example under-the-hood sandbox invocation:
bwrap \
  --ro-bind /usr /usr \
  --ro-bind /lib /lib \
  --ro-bind /nix /nix \
  --dev /dev \
  --proc /proc \
  --bind /mnt/user/media /media \
  --bind /mnt/user/appdata/radarr /config \
  --uid 99 --gid 100 \
  nix run nixpkgs#radarr
```
The `--uid 99` and `--gid 100` flags spoof the user inside the container to match Unraid's `nobody:users` group, ensuring files written to your shares have the correct permissions.

## 4. Running Rust Unit Tests

All business logic (parsers, sandbox generators, API clients) has comprehensive unit test coverage. To execute the test suite:

1. Enter the source directory:
   ```bash
   cd src/
   ```
2. Run cargo tests:
   ```bash
   cargo test
   ```

Tests cover:
* Config file validation and YAML output checks.
* `bubblewrap` CLI arguments builder matching.
* Nix search JSON result mapping.
* Path validity constraints (preventing store loops on `/boot`).
